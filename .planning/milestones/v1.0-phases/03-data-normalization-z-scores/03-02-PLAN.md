  <action>
    Create the engine module that orchestrates the full normalization pipeline and generates status.json.

    **pipeline/normalize/engine.py:**

    1. **`process_indicator(name, config)`** -- Process a single indicator end-to-end:
       - Call `normalize_indicator(name, config)` from ratios.py to get normalized quarterly DataFrame
       - If result is None (missing data), return None
       - Call `compute_rolling_zscores(df)` from zscore.py to add Z-scores
       - Get the latest valid row (last row with non-NaN z_score)
       - If no valid Z-scores (all NaN due to insufficient data), return None
       - Apply polarity from weights config: `oriented_z = z * polarity`
       - Compute gauge value: `zscore_to_gauge(oriented_z)`
       - Classify zone: `classify_zone(gauge_value)`
       - Determine confidence: `determine_confidence(window_size)`
       - Compute staleness: days between latest data_date and today
       - Extract history: last 12 valid gauge values from the Z-score series (convert each Z to gauge)
       - Return dict with all gauge metadata (matching the status.json per-gauge schema)

    2. **`build_gauge_entry(name, latest_row, z_df, weight_config)`** -- Build a single gauge dict:
       - Extract from latest_row: z_score, rolling_median, rolling_mad, window_size, value (raw normalized)
       - Apply polarity and compute gauge value
       - Build history array: take last 12 rows of z_df that have valid z_scores, convert each to gauge value
       - Return dict:
         ```python
         {
             "value": round(gauge_value, 1),
             "zone": zone_id,
             "zone_label": zone_label,
             "z_score": round(oriented_z, 3),
             "raw_value": round(latest_row['value'], 2),
             "raw_unit": "% YoY",  # or from config
             "weight": weight_config['weight'],
             "polarity": weight_config['polarity'],
             "data_date": latest_row['date'].strftime('%Y-%m-%d'),
             "staleness_days": (datetime.now() - latest_row['date']).days,
             "confidence": confidence,
             "interpretation": generate_interpretation(name, zone_id, latest_row['value']),
             "history": history_array  # list of up to 12 floats (gauge values)
         }
         ```

    3. **`generate_interpretation(indicator_name, zone, raw_value)`** -- Generate plain-text interpretation:
       - Based on indicator name and zone, return a human-readable string
       - Examples:
         * inflation + hot: "Inflation significantly above long-run average"
         * inflation + neutral: "Inflation near long-run average"
         * employment + warm: "Labour market tighter than historical average"
         * wages + cool: "Wage growth below long-run average"
       - Keep interpretations factual ("Data, not opinion") -- no recommendations
       - Return: str

    4. **`generate_status()`** -- Main entry point that generates status.json:
       - Load weights from weights.json via `load_weights(WEIGHTS_FILE)`
       - Process each indicator in INDICATOR_CONFIG:
         * Call process_indicator for each
         * Collect successful results into gauges dict
         * Track missing/failed indicators
       - Also try OPTIONAL_INDICATOR_CONFIG indicators (will return None for stubs)
       - Compute hawk score: `compute_hawk_score(gauge_values, weights)`
       - Classify overall zone and generate verdict
       - Determine overall confidence (lowest confidence among available gauges)
       - Build the complete status dict:
         ```python
         {
             "generated_at": datetime.utcnow().isoformat() + 'Z',
             "pipeline_version": "1.0.0",
             "overall": {
                 "hawk_score": round(hawk_score, 1),
                 "zone": overall_zone,
                 "zone_label": overall_zone_label,
                 "verdict": verdict,
                 "confidence": overall_confidence
             },
             "gauges": gauges_dict,
             "weights": {name: cfg['weight'] for name, cfg in weights.items()},
             "metadata": {
                 "window_years": ZSCORE_WINDOW_YEARS,
                 "clamp_range": [ZSCORE_CLAMP_MIN, ZSCORE_CLAMP_MAX],
                 "mapping": "linear",
                 "statistics": "robust (median/MAD)",
                 "indicators_available": len(gauges_dict),
                 "indicators_missing": missing_indicators
             }
         }
         ```
       - Create output directory: `STATUS_OUTPUT.parent.mkdir(parents=True, exist_ok=True)`
       - Write to STATUS_OUTPUT with `json.dump(status, f, indent=2)`
       - Print summary: number of gauges, hawk score, verdict
       - Return the status dict (for testing)

    5. **`__main__` block:**
       - When run directly (`python -m pipeline.normalize.engine`), call generate_status()
       - Print the generated status.json path
       - Useful for development and testing without running the full pipeline
