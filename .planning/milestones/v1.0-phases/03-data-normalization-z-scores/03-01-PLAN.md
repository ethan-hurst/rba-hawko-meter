<objective>
Build the normalization engine: ratio calculations, robust Z-score computation, gauge mapping, and configurable weights.

Purpose: Creates the three-step statistical pipeline that transforms raw economic CSV data into comparable 0-100 gauge values. Step 1: raw values to ratios via YoY percentage change. Step 2: ratios to robust Z-scores via 10-year rolling median/MAD. Step 3: Z-scores to 0-100 gauge values via [-3,+3] linear clamp. Also establishes weights.json with documented expert-judgment weights.

Output: Three Python modules (ratios.py, zscore.py, gauge.py), indicator config in config.py, and data/weights.json -- ready for Plan 02 to wire into the engine and produce status.json.
</objective>

  <action>
    **1. Create pipeline/normalize/zscore.py** -- Robust Z-score calculation:

    - `calculate_mad(values)`: Median Absolute Deviation with 1.4826 normal-consistency scaling. Formula: `np.median(np.abs(values - np.median(values))) * 1.4826`. Return 0.0 if all values identical.

    - `robust_zscore(current_value, window_values)`: Single Z-score. `z = (current - median) / MAD`. Return 0.0 if MAD == 0.

    - `compute_rolling_zscores(df, window_quarters=40, min_quarters=20)`: For each row, compute robust Z-score against look-back window (exclude current value from window). Rows before min_quarters get NaN. Add columns: z_score, rolling_median, rolling_mad, window_size.

    - `determine_confidence(window_size)`: >= 32 obs -> "HIGH", >= 20 -> "MEDIUM", else "LOW".

    **Key constraints from CONTEXT.md:**
    - Use Median/MAD (robust statistics), NOT Mean/StdDev -- per locked decision
    - 10-year rolling window (40 quarters) with 5-year minimum (20 quarters) -- per locked decision
    - Zero dispersion (MAD=0) returns Z=0 (neutral) -- no variability means no pressure signal

    **2. Create pipeline/normalize/gauge.py** -- Gauge mapping and scoring:

    - `zscore_to_gauge(z, clamp_min=-3.0, clamp_max=3.0)`: Linear map: `gauge = ((z - clamp_min) / (clamp_max - clamp_min)) * 100`, clip to [0, 100]. Return NaN if input is NaN.

    - `classify_zone(gauge)`: 0-20 -> ("cold", "Strong dovish pressure"), 20-40 -> ("cool", "Mild dovish pressure"), 40-60 -> ("neutral", "Balanced"), 60-80 -> ("warm", "Mild hawkish pressure"), 80-100 -> ("hot", "Strong hawkish pressure"). NaN -> ("unknown", "Insufficient data").

    - `apply_polarity(z, polarity)`: `z * polarity`. Supports +1 (standard) and -1 (inverted for future indicators).

    - `load_weights(path)`: Load weights.json, validate all weights > 0, validate sum is 0.99-1.01. Raise ValueError on failure.

    - `compute_hawk_score(gauge_values, weights, exclude_benchmark=True)`: Weighted average of available gauges. If exclude_benchmark, skip 'asx_futures'. Rebalance remaining weights to sum to 1.0 when indicators are missing. Clip result to [0, 100].

    - `generate_verdict(hawk_score)`: 0-20 -> strong easing, 20-40 -> mild easing, 40-60 -> balanced, 60-80 -> moderate tightening, 80-100 -> strong tightening.

    **Key constraints from CONTEXT.md:**
    - Linear mapping (not sigmoid/CDF) -- per locked decision
    - 5 zones: Cold/Cool/Neutral/Warm/Hot -- per locked decision
    - Configurable weights in weights.json -- per locked decision
    - [-3, +3] clamp range per research refinement (original was [-2, +2], widened to reduce clipping from 4.6% to 0.3%)
  </action>
