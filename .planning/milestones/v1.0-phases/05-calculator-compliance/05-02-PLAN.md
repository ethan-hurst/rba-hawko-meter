<task type="auto">
  <name>Task 1: Audit all HTML content for ASIC compliance and neutral language</name>
  <files>
    public/index.html
  </files>
  <action>
    Perform a comprehensive audit of every piece of text in public/index.html against ASIC RG 244 requirements. This task is the core compliance review.

    **Audit checklist -- apply the four compliance tests to EVERY text element:**

    1. **"Data Not Opinion" Test:** Does this text present verifiable data, or express a view?
    2. **"Action Implication" Test:** Could this text be read as suggesting the user do something?
    3. **"Reasonable Person" Test:** Could a reasonable person regard this as intended to influence a financial decision?
    4. **"Educational Framing" Test:** Is this framed as learning/exploration, not advice?

    **Specific items to audit and fix:**

    a. **Disclaimer banner** (top of page):
       - Must be present and visible without scrolling
       - Must state "General Information Only"
       - Must NOT be removable from DOM (CSS hide is OK, DOM removal is not)
       - Verify wording aligns with research recommendations

    b. **Header and subtitle:**
       - Title "RBA Hawk-O-Meter" -- OK (descriptive)
       - Subtitle "Data, not opinion." -- OK (reinforces factual framing)
       - Check for any taglines or descriptions that could imply advice

    c. **Hero/gauge section:**
       - Check any gauge-related text
       - Ensure hawk score is described as "data aggregation" not "prediction"
       - Verify colour descriptions use "Hawkish"/"Dovish" not "Danger"/"Safe"
       - Check for any text like "the RBA will..." -- replace with "market pricing implies..."

    d. **Countdown section:**
       - "Next RBA Board Meeting" -- OK (factual)
       - Check for any text implying what will happen at the meeting
       - "Meeting TODAY" banner should be purely informational, not "rates will change today"

    e. **Chart section:**
       - Source citation present: "Source: Reserve Bank of Australia"
       - Chart title and axis labels are neutral
       - Rate change annotations should say "Rate changed to X%" not "Rate hike/cut warning"

    f. **Individual gauge sections (Phase 4):**
       - Each gauge interpretation text must be factual: "Prices rising faster than wages" (factual) not "This is bad for borrowers" (opinion)
       - Verdict text: "HAWKISH -- Multiple indicators suggest upward rate pressure" (factual) not "Warning: rates likely to rise" (opinion/advice)
       - Source citations on each gauge data point

    g. **ASX Futures section (Phase 4):**
       - Must say "Market-implied probability" not "Prediction"
       - Source: "Derived from ASX 30 Day Interbank Cash Rate Futures"
       - Must NOT say "rates will..." -- say "market pricing implies..."

    h. **Calculator section (Phase 5 Plan 01):**
       - Heading: "Explore Rate Scenarios" -- verify
       - Intro: "See how different interest rates could affect a typical Australian mortgage" -- verify "typical" not "your"
       - Calculator disclaimer: "Illustrative Estimates Only" -- verify present and correct
       - Input labels: neutral, no implied advice
       - Results text: "Estimated repayments would be..." not "Your repayments will be..."
       - Comparison text: factual differences, no "you should consider..."

    i. **Footer disclaimer** -- must contain ALL of these elements:
       1. "General Information Only" heading
       2. Statement: site shows factual economic data for educational purposes
       3. Statement: not financial advice, recommendation, or forecast
       4. Statement: performs mathematical computations on publicly available data
       5. Statement: does not take into account personal objectives, financial situation, or needs
       6. Recommendation: speak to a licensed financial adviser before making financial decisions
       7. Statement: creators do not hold an Australian Financial Services Licence
       8. Data sources section with proper attribution:
          - "Source: Reserve Bank of Australia" (CC BY 4.0)
          - "Source: Australian Bureau of Statistics" (CC BY 2.5 AU)
          - "Derived from ASX 30 Day Interbank Cash Rate Futures"
          - Note: "These organisations do not endorse this website"

    j. **Words/phrases to search and replace across ALL content:**
       - "prediction" -> "market expectation" or "market pricing"
       - "forecast" -> "market expectation"
       - "your mortgage" -> "a mortgage" or "the mortgage"
       - "your repayments" -> "the repayments" or "estimated repayments"
       - "you should" -> remove or rephrase as factual
       - "we recommend" -> remove or rephrase
       - "will rise" / "will fall" -> "market pricing implies" or "could"
       - "danger" / "warning" / "safe" (in financial context) -> "hawkish" / "dovish" / "neutral"
       - "lock in" / "act now" -> remove entirely

    **Document all changes** in a comment at the top of the commit or summary.
  </action>
  <verify>
    Search entire index.html for risky language:
    `grep -i "prediction\|forecast\|you should\|we recommend\|your mortgage\|your repayment\|will rise\|will fall\|danger\|warning\|lock in\|act now" public/index.html`
    Expected: NO matches (all risky language removed).

    Verify disclaimer present:
    `grep "General Information Only" public/index.html` returns at least 2 matches (banner + footer).
    `grep "Australian Financial Services Licence" public/index.html` returns a match.
    `grep "Illustrative Estimates Only" public/index.html` returns a match.

    Verify source citations:
    `grep "Reserve Bank of Australia" public/index.html` returns matches.

    Verify educational framing:
    `grep "typical mortgage\|typical Australian" public/index.html` returns matches.
    `grep -c "your mortgage" public/index.html` returns 0.
  </verify>
  <done>
    Every text element in index.html has been audited against the four ASIC compliance tests. All risky language replaced with neutral alternatives. Disclaimers verified complete and correct. Source citations present for all data points. Educational framing consistent throughout. No content constitutes financial advice.
  </done>

<task type="auto">
  <name>Task 3: Final accessibility audit of calculator and dashboard</name>
  <files>
    public/index.html
    public/js/calculator.js
  </files>
  <action>
    Perform a targeted accessibility audit focusing on the calculator (new in Phase 5) and verifying existing Phase 2/4 accessibility.

    **Calculator accessibility checks:**

    a. **Form inputs:**
       - Every `<input>` and `<select>` has an associated `<label>` with matching `for` attribute
       - Help text connected via `aria-describedby`
       - Required fields have `required` attribute and `aria-required="true"`
       - Error states announced via `aria-live="polite"` region
       - Custom validation messages set via `setCustomValidity()` (accessible to screen readers)

    b. **Scenario slider:**
       - `<input type="range">` has a `<label>`
       - Paired with `<output for="...">` element
       - `aria-valuemin`, `aria-valuemax`, `aria-valuenow`, `aria-valuetext` all present
       - `aria-valuetext` provides human-readable value ("3.85 percent" not just "3.85")
       - Output has `aria-live="polite"` for dynamic updates
       - Focus indicator visible (no outline:none)

    c. **Comparison table:**
       - Uses semantic `<table>` with `<caption>`
       - `<thead>` with `<th scope="col">` for column headers
       - `<tbody>` with `<th scope="row">` for row headers (if applicable)
       - Table is navigable with screen reader table navigation

    d. **Results area:**
       - Dynamic results in an `aria-live="polite"` region
       - Color coding is NOT the only way to convey information (text also indicates +/-)
       - Sufficient color contrast for all result text (4.5:1 minimum)

    e. **General page checks:**
       - Heading hierarchy: h1 -> h2 -> h3 (no skipped levels)
       - All sections have appropriate aria-labels
       - Skip navigation link or logical heading structure for keyboard navigation
       - Focus management: no focus traps, logical tab order

    **Fix any issues found.** If issues require changes to JS, update the corresponding file.
  </action>
  <verify>
    Verify all form inputs have labels:
    `grep -c "<label" public/index.html` should be >= 5 (one per calculator input plus others).
    `grep "aria-describedby" public/index.html` returns matches for help text connections.

    Verify slider accessibility:
    `grep "aria-valuenow\|aria-valuetext\|aria-live" public/index.html` returns matches.
    `grep "<output" public/index.html` returns at least one match.

    Verify comparison table semantics:
    `grep "<caption\|<thead\|<th" public/index.html` returns matches.

    Verify no outline removal:
    `grep "outline.*none\|outline: 0" public/index.html public/js/*.js` returns NO matches (or only in reset styles with alternative focus indicators).

    Verify color is not sole indicator:
    Visual inspection: difference values show both "+$X" / "-$X" text AND color.
  </verify>
  <done>

Results area uses aria-live for dynamic updates. Color is never the sole information carrier. Focus indicators visible on all interactive elements.
  </done>
