  <name>Task 2: Create calculator.js module with repayment math, comparison, localStorage, and form handling</name>
  <files>

  <action>
    Create the CalculatorModule using the IIFE pattern (consistent with Phase 2 modules: DataModule, ChartModule, CountdownModule).

    **public/js/calculator.js -- CalculatorModule:**

    IIFE pattern exposing a public API. All financial math uses Decimal.js (global `Decimal` from CDN).

    **Private constants:**
    ```javascript
    const STORAGE_KEY = 'rba-hawko-calculator';
    const DEFAULTS = {
      loanAmount: 600000,
      termYears: 25,
      interestRate: 3.85,
      repaymentType: 'PI',
      frequency: 'monthly'
    };
    const FREQUENCY_LABELS = {
      monthly: { label: 'per month', divisor: 1, perYear: 12 },
      fortnightly: { label: 'per fortnight', divisor: 2, perYear: 26 },
      weekly: { label: 'per week', divisor: 4, perYear: 52 }
    };
    ```

    **Private functions:**

    1. `calculateMonthlyPaymentPI(principal, annualRatePct, termYears)`:
       - Convert all inputs to Decimal via string: `new Decimal(principal.toString())`
       - Calculate monthly rate: `annualRatePct / 100 / 12`
       - If rate is 0 or very close to 0 (< 0.0001): return `principal / (termYears * 12)` (no interest case)
       - Apply amortization formula: `M = P * [r(1+r)^n] / [(1+r)^n - 1]`
       - Return Decimal object (caller formats with `.toFixed(2)`)

    2. `calculateMonthlyPaymentIO(principal, annualRatePct)`:
       - Convert to Decimal
       - Calculate: `P * (annualRate / 100 / 12)`
       - Return Decimal object

    3. `calculatePayment(inputs)`:
       - Dispatch to PI or IO based on inputs.repaymentType
       - Get monthly payment from the appropriate formula
       - Adjust for frequency using FREQUENCY_LABELS[inputs.frequency].divisor
       - Return Decimal object for the per-period payment

    4. `calculateComparison(inputs, scenarioRate)`:
       - Calculate current payment using inputs.interestRate
       - Calculate scenario payment using scenarioRate
       - Calculate +0.25% payment using `inputs.interestRate + 0.25`
       - For each, compute: per-period payment, annual cost (payment * paymentsPerYear)
       - Return object:
         ```javascript
         {
           current: { rate, perPeriod, annual },
           scenario: { rate, perPeriod, annual },
           quarterPoint: { rate, perPeriod, annual },
           diff: {
             perPeriod: scenario.perPeriod - current.perPeriod,
             annual: scenario.annual - current.annual
           }
         }
         ```
       - All values as formatted strings (Decimal.toFixed(2))

    5. `formatCurrency(decimalValue)`:
       - Format with $ prefix and comma separators
       - Use `Intl.NumberFormat('en-AU', { style: 'currency', currency: 'AUD', minimumFractionDigits: 2 })`
       - Input: Decimal object or string. Output: formatted string like "$3,104.23"

    6. `formatDifference(decimalValue)`:
       - Format as signed currency: "+$763.44" or "-$200.10"
       - Positive values get red color class, negative get green, zero gets neutral
       - Return { text: string, className: string }

    7. `saveInputs(inputs)`:
       - Validate inputs is a non-null object
       - try { localStorage.setItem(STORAGE_KEY, JSON.stringify(inputs)); return true; }
       - catch: log warning, return false (never crash)

    8. `loadInputs()`:
       - try { parse localStorage.getItem(STORAGE_KEY) }
       - Validate parsed data: all expected fields present with correct types
       - Validate ranges: loanAmount 1000-10000000, termYears 1-30, interestRate 0.01-15, repaymentType in ['PI','IO'], frequency in ['monthly','fortnightly','weekly']
       - If invalid or missing: return { ...DEFAULTS }
       - catch: remove corrupted key, return { ...DEFAULTS }

    9. `isValidInputs(data)`:
       - Check all fields exist and have correct types
       - Check all values within valid ranges
       - Return boolean

    10. `updateDisplay(comparison, frequency)`:
        - Use safe DOM methods (createElement/textContent) -- NO innerHTML
        - Update current payment display: id="calc-current-payment"
        - Update scenario payment display: id="calc-scenario-payment"
        - Update difference displays: id="calc-diff-per-period", id="calc-diff-annual"
        - Update comparison table rows
        - Update frequency labels (per month/fortnight/week)
        - Update slider output text
        - Apply red/green color classes to difference values

    11. `updateSliderOutput(slider, output)`:
        - Set output.textContent to `${slider.value}%`
        - Set slider aria-valuenow and aria-valuetext

    12. `getInputsFromForm()`:
        - Read values from all 5 form fields
        - Parse numbers from string inputs
        - Return inputs object matching DEFAULTS shape

    **Public API:**

    1. `init(containerSelector)`:
       - Load saved inputs from localStorage (or defaults)
       - Populate form fields with loaded values
       - Set slider value to current interest rate
       - Perform initial calculation and display
       - Attach event listeners:
         * All 5 form inputs: 'change' event -> recalculate, save to localStorage
         * Number inputs: 'input' event with debounce (300ms) -> recalculate, save
         * Slider: 'input' event (no debounce -- must be instant for live feel) -> recalculate
         * Slider: 'change' event -> save to localStorage (don't save on every slide)
       - Attach rate sync: when "Current Interest Rate" input changes, update slider default position
       - Return true if initialized successfully

    2. `recalculate()`:
       - Read inputs from form
       - Validate inputs (if invalid, show inline error, don't crash)
       - Get scenario rate from slider
       - Call calculateComparison(inputs, scenarioRate)
       - Call updateDisplay(comparison, inputs.frequency)

    **Important implementation notes:**
    - IIFE pattern: `const CalculatorModule = (() => { ... })();`
    - All Decimal operations use string conversion: `new Decimal(val.toString())`
    - NO innerHTML anywhere -- use createElement/textContent for all DOM updates
    - Debounce helper for number input events (inline, not imported)
    - Handle edge case: 0% interest rate (pure principal division)
    - Handle edge case: IO at 0% rate (payment = $0)
    - Slider step of 0.25 matches RBA's typical move size
    - Form fields use HTML5 validation attributes as a first line of defense
    - JavaScript validation as second line (for edge cases)
  </action>
