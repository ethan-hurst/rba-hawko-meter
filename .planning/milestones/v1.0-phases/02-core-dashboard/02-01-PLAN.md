  <name>Task 1: Create frontend data generator script</name>
  <files>

  <action>
    Create a Python script that transforms the pipeline's CSV output into frontend-consumable JSON files.

    **scripts/generate_frontend_data.py:**

    1. **rates.json** -- Transform data/rba_cash_rate.csv into chart-ready format:
       - Read the CSV (columns: date, value, source)
       - Sort by date ascending
       - Detect rate changes: compare consecutive rows, where value changes record {date, previous_rate, new_rate, direction: "up"|"down", amount: abs(change)}
       - Output structure:
         ```json
         {
           "last_updated": "2026-02-04",
           "current_rate": 4.35,
           "source": "RBA",
           "history": {
             "dates": ["2015-01-01", "2015-01-02", ...],
             "rates": [2.50, 2.50, ...]
           },
           "rate_changes": [
             {"date": "2022-05-04", "from": 0.10, "to": 0.35, "direction": "up", "amount": 0.25},
             ...
           ]
         }
         ```
       - Write to public/data/rates.json with indent=2

    2. **meetings.json** -- Generate RBA meeting schedule:
       - RBA Board meets on the first Tuesday of each month (except January)
       - Decision time is 2:30pm AEST (14:30+11:00, or 14:30+10:00 during AEDT)
       - Generate meetings for current year and next year
       - Identify the next upcoming meeting from today's date
       - Output structure:
         ```json
         {
           "next_meeting": {
             "date": "2026-03-03T14:30:00+11:00",
             "display_date": "3 March 2026",
             "display_time": "2:30pm AEST"
           },
           "meetings_2026": [
             {"date": "2026-02-03T14:30:00+11:00", "display_date": "3 February 2026"},
             ...
           ]
         }
         ```
       - Use zoneinfo (Python 3.9+ stdlib) for Australia/Sydney timezone
       - Handle AEST/AEDT correctly: store with correct UTC offset for each meeting date
       - Write to public/data/meetings.json with indent=2

    3. **Main block:**
       - Create public/data/ directory if it doesn't exist
       - Check if data/rba_cash_rate.csv exists; if not, print warning and generate meetings.json only (allow dashboard development without pipeline data)
       - Print summary: files created, current rate, next meeting date

    4. **public/data/.gitkeep** -- empty file

    **Important considerations:**
    - The script must work even if Phase 1 hasn't run yet (graceful fallback for meetings.json)
    - Use pathlib.Path for all file paths
    - Import from pipeline.config if available for DATA_DIR, but fall back to hardcoded "data" path
    - Round rate values to 2 decimal places in JSON output
  </action>

  </verify>
