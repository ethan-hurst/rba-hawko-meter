  <name>Task 2: Rewrite asx_futures_scraper.py to use JSON endpoints</name>

  <action>
  Replace the entire contents of `pipeline/ingest/asx_futures_scraper.py` with a new implementation. Preserve the module-level docstring theme and the `fetch_and_save()` function signature exactly (CI depends on it).

  The new implementation has three internal functions plus the public `fetch_and_save()`:

  **1. `_fetch_json(session, url, label)` helper:**
  - Takes a requests.Session, URL string, and label string for logging
  - Calls `session.get(url, timeout=DEFAULT_TIMEOUT)`
  - Calls `response.raise_for_status()`
  - Returns `response.json()` (the endpoints return JSON despite .csv extension)
  - On failure, logs error and re-raises

  **2. `_derive_probabilities(implied_rate, current_rate)` helper:**
  - Computes `change_bp = round((implied_rate - current_rate) * 100, 1)`
  - Applies the probability derivation logic from research:
    - If `change_bp < -5`: probability_cut = min(100, round(abs(change_bp) / 25 * 100)), probability_hold = 100 - probability_cut, probability_hike = 0
    - If `change_bp > 5`: probability_hike = min(100, round(change_bp / 25 * 100)), probability_hold = 100 - probability_hike, probability_cut = 0
    - Else (within +/-5bp): probability_hold = 100, probability_cut = 0, probability_hike = 0
  - Returns tuple: `(change_bp, probability_cut, probability_hold, probability_hike)`

  **3. `scrape_asx_futures()` main scrape function:**
  - Creates session via `create_session(retries=3, backoff_factor=0.5, user_agent=BROWSER_USER_AGENT)`
  - Fetches `ASX_FUTURES_URLS["dynamic_text"]` via `_fetch_json()` to get `currentCashRate` (float)
  - Fetches `ASX_FUTURES_URLS["market_expectations"]` via `_fetch_json()` to get the meeting expectations array
  - `scrape_date = datetime.now().strftime('%Y-%m-%d')`
  - Iterates over each meeting entry in the expectations array. Each entry is expected to have keys like `meetingDate` (string, e.g. "18 Feb 2026") and `impliedRate` or `ExpectedRate` (float, percentage). Log the raw keys of the first entry at DEBUG level so future debugging can identify key names if the schema changes.
  - For each meeting entry, parse the meeting date string to `YYYY-MM-DD` format using `datetime.strptime` with appropriate format (try `"%d %b %Y"` first, fall back to `"%Y-%m-%d"` if already ISO format). Skip entries where date parsing fails (log warning, continue).
  - Extract the implied rate. Try keys in order: `impliedRate`, `ExpectedRate`, `expectedRate`, `rate`. The rate may be a percentage (e.g. 4.10) or a string — convert to float. Skip entry if no rate key found (log warning).
  - Call `_derive_probabilities(implied_rate, current_rate)` for each meeting
  - Build a list of row dicts with keys: `date`, `meeting_date`, `implied_rate`, `change_bp`, `probability_cut`, `probability_hold`, `probability_hike`
  - Return `pd.DataFrame(rows)` — empty DataFrame if no rows extracted
  - Validate: all implied_rate values should be between 0 and 15 (log warning if not, but don't filter)

  **4. `fetch_and_save()` public interface (signature unchanged):**
  - Calls `scrape_asx_futures()`
  - If DataFrame is empty, return `{'status': 'failed', 'error': 'No data extracted from ASX endpoints'}`
  - Otherwise, `output_path = DATA_DIR / "asx_futures.csv"` and write the CSV directly (do NOT use `append_to_csv` — its dedup on `date` alone would discard multiple meetings per scrape date):
    - If `output_path` exists, read existing CSV with `pd.read_csv(output_path)`, concatenate with new `df`, then `drop_duplicates(subset=['date', 'meeting_date'], keep='last')` to dedup on the composite key
    - If `output_path` does not exist, just use `df` as-is
    - Write the result with `result_df.to_csv(output_path, index=False)`
  - Return `{'status': 'success', 'rows': len(result_df), 'meetings': len(df)}`
  - Wrap entire body in try/except, on exception: `logger.error(f"ASX scraper failed: {e}")`, return `{'status': 'failed', 'error': str(e)}`

  **5. `if __name__ == '__main__':` block:**
  - Print header, call `fetch_and_save()`, print result
  - If success, read and print first 5 rows of output CSV
