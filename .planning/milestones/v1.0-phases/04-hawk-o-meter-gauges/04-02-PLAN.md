    New public method:

    3. **generateMetricInterpretation(metricId, metricData)** — returns plain-text string from actual data values.
       - metricData: { value, z_score, raw_value, raw_unit, data_date, staleness_days, confidence, interpretation, history }
       - Note: status.json already provides an `interpretation` string per gauge. Use it as fallback for unknown metricIds.
       - For known metricIds, generate richer text using raw_value:

       Use a switch/map on metricId to generate metric-specific text with real numbers:

       **inflation**: Read raw_value (CPI % YoY), compare to 2-3% target band.
         - value < 40: "CPI at {raw_value}{raw_unit} — within or below 2-3% target band"
         - value 40-60: "CPI at {raw_value}{raw_unit} — near top of 2-3% target band"
         - value > 60: "CPI at {raw_value}{raw_unit} — above 2-3% target band"

       **wages**: Read raw_value (WPI % YoY).
         - value < 40: "Wages growing {raw_value}{raw_unit} — subdued growth"
         - value 40-60: "Wages growing {raw_value}{raw_unit} — moderate growth"
         - value > 60: "Wages growing {raw_value}{raw_unit} — strong growth adding to cost pressure"

       **building_approvals**: Read raw_value (% YoY change in approvals).
         - value < 40: "Building approvals at {raw_value}{raw_unit} — below trend, easing construction pressure"
         - value 40-60: "Building approvals at {raw_value}{raw_unit} — near long-run average"
         - value > 60: "Building approvals at {raw_value}{raw_unit} — above trend, adding to demand pressure"

       **housing**: Read raw_value (price-to-income ratio or % YoY).
         - value < 40: "Housing prices at {raw_value}{raw_unit} — easing affordability pressure"
         - value 40-60: "Housing prices at {raw_value}{raw_unit} — stable"
         - value > 60: "Housing prices at {raw_value}{raw_unit} — elevated affordability pressure"

       **jobs** (maps from "employment" in status.json): Read raw_value.
         - value < 40: "Labour market at {raw_value}{raw_unit} — softening conditions"
         - value 40-60: "Labour market at {raw_value}{raw_unit} — balanced conditions"
         - value > 60: "Labour market at {raw_value}{raw_unit} — tight conditions adding to wage pressure"

       **spending**: Read raw_value (real retail turnover % change).
         - value < 40: "Consumer spending at {raw_value}{raw_unit} — subdued demand"
         - value 40-60: "Consumer spending at {raw_value}{raw_unit} — moderate demand"
         - value > 60: "Consumer spending at {raw_value}{raw_unit} — strong demand adding to price pressure"

       **business_confidence** (maps from "capacity" display label): Read raw_value.
         - value < 40: "Business confidence at {raw_value}{raw_unit} — below average"
         - value 40-60: "Business confidence at {raw_value}{raw_unit} — near long-term average"
         - value > 60: "Business confidence at {raw_value}{raw_unit} — above average"

       **Default fallback**: Use metricData.interpretation from status.json (Phase 3 already provides this). If that's also missing, fall back to "{label}: {raw_value}{raw_unit}"

       All text uses neutral, factual framing. No "you should" or "expect" language.
       Numbers formatted via: parseFloat(raw_value).toFixed(1) — no Intl dependency for brevity.

    4. **renderMetricCard(containerId, metricId, metricData, weight)** — builds a complete metric card:
       - Create card wrapper div: bg-finance-gray rounded-lg p-4 border border-finance-border
       - Derive stale from: metricData.staleness_days > 90
       - If stale: add 'border-amber-500/50' class to card
       - If metricData.confidence === 'LOW': add small amber badge "(low confidence)"
       - Look up display label from DISPLAY_LABELS map (defined in gauges.js createBulletGauge). Fallback: capitalize and replace underscores
       - Create header row: label (h4, font-semibold, text-gray-200) + weight badge (text-xs, text-gray-500, "XX% weight")
         - Weight from metricData.weight (already present in status.json per gauge)
       - Create gauge container div with unique id: `gauge-${metricId}`
       - Create interpretation text div: text-sm, text-gray-400, id: `interp-${metricId}`
       - Set interpretation text via textContent = generateMetricInterpretation(metricId, metricData)
       - Create source citation: text-xs, text-gray-600, "Data as of {data_date}" (status.json has data_date, not source)
       - Append all children to card, append card to container
       - All DOM via createElement/textContent — no innerHTML
