  <action>
    Create two IIFE modules: one for Plotly gauge rendering, one for verdict/table display.

    **public/js/gauges.js — GaugesModule IIFE:**

    Private constants and helpers:

    1. **ZONE_COLORS** — array of 5 zone objects:
       - `{ range: [0, 20], color: '#1e40af', label: 'STRONGLY DOVISH' }` (deep blue)
       - `{ range: [20, 40], color: '#60a5fa', label: 'DOVISH' }` (light blue)
       - `{ range: [40, 60], color: '#6b7280', label: 'NEUTRAL' }` (grey)
       - `{ range: [60, 80], color: '#f87171', label: 'HAWKISH' }` (light red)
       - `{ range: [80, 100], color: '#dc2626', label: 'STRONGLY HAWKISH' }` (deep red)
       Note: Uses Blue/Grey/Red instead of CONTEXT.md's Green/Amber/Red per 04-RESEARCH.md
       colorblind accessibility recommendation (8% of males have red-green deficiency).

    2. **getZoneColor(value)** — returns color for the zone `value` falls in. Clamp to first/last zone for out-of-range.

    3. **getGaugeSteps()** — returns Plotly steps array derived from ZONE_COLORS: `[{ range: [0,20], color: '#1e40af' }, ...]`

    4. **getDarkLayout(overrides)** — returns Plotly layout for dark theme:
       - paper_bgcolor: 'transparent', plot_bgcolor: 'transparent'
       - font: { color: '#e5e5e5', family: 'Inter, system-ui, sans-serif' }
       - margin: { t: 40, r: 25, l: 25, b: 0 }
       - Shallow-merge overrides via Object.assign

    Public methods:

    5. **getStanceLabel(value)** — returns stance string for the zone `value` falls in.

    6. **createHeroGauge(containerId, hawkScore)** — creates large semicircle gauge:
       - Trace: type 'indicator', mode 'gauge+number', value hawkScore
       - title: { text: getStanceLabel(hawkScore), font: { size: 20, color: getZoneColor(hawkScore) } }
       - number: { font: { size: 52, color: '#f3f4f6' }, valueformat: '.0f' }
       - gauge.shape: 'angular' (semicircle is default for angular)
       - gauge.axis: range [0,100], tickwidth 1, tickcolor '#4a4a4a', tickfont { size: 12, color: '#9ca3af' }
       - gauge.bar: { color: getZoneColor(hawkScore), thickness: 0.6 }
       - gauge.bgcolor: '#1f2937', gauge.borderwidth: 0
       - gauge.steps: getGaugeSteps()
       - gauge.threshold: { line: { color: '#fbbf24', width: 3 }, thickness: 0.75, value: 50 } (neutral reference)
       - domain: { x: [0,1], y: [0,1] }
       - Layout: getDarkLayout() with NO fixed width/height (container CSS handles sizing)
       - Config: { responsive: true, displayModeBar: false }
       - Call Plotly.newPlot(containerId, [trace], layout, config)

    7. **updateHeroGauge(containerId, hawkScore)** — efficient re-render via Plotly.react()

    **public/js/interpretations.js — InterpretationsModule IIFE:**

    1. **renderVerdict(containerId, overallData)** — overallData is `data.overall` from status.json
       - Clear container with textContent = ''
       - Extract stance from overallData.zone_label. Map to display label:
         { "Strong dovish pressure": "STRONGLY DOVISH", "Mild dovish pressure": "DOVISH", "Balanced": "NEUTRAL", "Mild hawkish pressure": "HAWKISH", "Strong hawkish pressure": "STRONGLY HAWKISH" }
       - Create label span: bold, colored via GaugesModule.getZoneColor(overallData.hawk_score)
       - Create text span: " — " + overallData.verdict (string), text-gray-300
       - Fallback: if overallData is a plain string, display it directly
       - Append both via safe DOM methods

    2. **renderASXTable(containerId, asxData)** — asxData: { current_rate, implied_rate, probabilities: {cut,hold,hike}, source, source_date, next_meeting }
       - Clear container
       - Build heading: "ASX Futures — Next Meeting" (h3, font-semibold, text-gray-200)
       - Build table with thead (Outcome | Probability) and 3 tbody rows:
         * "Rate Cut (-0.25%)" — text colored #60a5fa (blue)
         * "Hold (unchanged)" — text colored #9ca3af (grey)
         * "Rate Hike (+0.25%)" — text colored #f87171 (red)
       - Highest probability row gets brighter text + subtle left border highlight
       - Format probabilities with Intl.NumberFormat('en-AU', { style: 'percent', minimumFractionDigits: 1 })
       - Below table: "Implied rate: X.XX%" in small text, source citation in xs grey
       - Handle null/missing probabilities: show "Data unavailable"
       - All DOM via createElement/textContent — no innerHTML

    3. **renderStalenessWarning(containerId, lastUpdated)** — ISO 8601 string
       - If > 7 days old: amber warning text
       - If <= 7 days: subdued grey "Updated X days ago" text

    Both modules must use IIFE pattern: `const XxxModule = (() => { ... return { publicMethod }; })();`
    No innerHTML anywhere — all DOM manipulation via createElement, textContent, appendChild.
