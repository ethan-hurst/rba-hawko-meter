<task type="auto">
  <name>Task 1: Install Lefthook and create pre-push hook configuration</name>
  <files>lefthook.yml, package.json</files>
  <action>
1. Install lefthook as npm devDependency:
   ```
   npm install @evilmartians/lefthook --save-dev
   ```
   Use `@evilmartians/lefthook` per user locked decision (research notes `lefthook` is the newer recommended package, but honoring the user's explicit choice).

2. Add `"postinstall": "lefthook install"` to scripts in package.json as belt-and-suspenders (lefthook's own postinstall already runs this, but makes intent explicit).

3. Create `lefthook.yml` at project root with this exact configuration:
   ```yaml
   # Show only failure output — silent on success
   output:
     - failure
     - execution_out

   pre-push:
     parallel: true
     commands:
       lint-py:
         run: ruff check pipeline/ tests/
         timeout: "30s"
         fail_text: "Push blocked: Python lint errors found. Run npm run test:fast to see details."
       lint-js:
         run: npx eslint public/js/
         timeout: "30s"
         fail_text: "Push blocked: JS lint errors found. Run npm run test:fast to see details."
       unit-tests:
         run: pytest tests/python/ -m "not live"
         timeout: "30s"
         fail_text: "Push blocked: Unit tests failed. Run npm run test:fast to see details."
   ```

   Key design decisions:
   - `parallel: true` at hook level runs all 3 commands concurrently — all failures shown together
   - `output: [failure, execution_out]` globally — silent on success, shows only failing output
   - `timeout: "30s"` per command as safety net (target is under 10s on clean codebase)
   - `fail_text` provides clear banner directing developer to `npm run test:fast`
   - Use `npx eslint` (not bare `eslint`) to ensure the locally-installed eslint binary is used, matching how `npm run lint:js` resolves it
   - `ruff` and `pytest` are available via pyenv shims on PATH

4. Verify the hook was installed by checking `.git/hooks/pre-push` exists.

5. Run `lefthook run pre-push` to confirm the hook executes successfully on the current clean codebase.
