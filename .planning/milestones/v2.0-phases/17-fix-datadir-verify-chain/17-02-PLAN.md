Update the `scripts` section of `package.json` to add individual tier scripts and wire verify_summary.py into the chain.

**Current scripts to update:**
```json

"test:fast": "npm run lint && pytest tests/python/ -m \"not live\""
```

**New scripts section (preserve all existing scripts, update verify and test:fast, add new ones):**

1. **Rename `test:fast` to use `python -m pytest` for consistency** (currently uses bare `pytest` which may not resolve correctly in all environments):
```json
"test:fast": "npm run lint && python -m pytest tests/python/ -m \"not live\""
```

2. **Add individual tier scripts:**
```json
"verify:fast": "npm run lint && python -m pytest tests/python/ -m \"not live\"",
"verify:live": "python -m pytest tests/python/ -m live -v",
"verify:playwright": "npx playwright test"
```

3. **Update verify to use tier scripts and include verify_summary.py:**
```json
"verify": "npm run verify:fast && npm run verify:live && npm run verify:playwright && python scripts/verify_summary.py"
```

The `&&` chaining ensures fail-fast behavior — if any tier fails, subsequent tiers and verify_summary.py do not run.

**Full expected scripts section after changes:**
```json
"scripts": {
  "postinstall": "lefthook install",
  "test": "npx playwright test",
  "test:headed": "npx playwright test --headed",
  "lint:py": "ruff check pipeline/ tests/",
  "lint:js": "npx eslint public/js/",
  "test:python:live": "python -m pytest tests/python/ -m live -v",
  "verify": "npm run verify:fast && npm run verify:live && npm run verify:playwright && python scripts/verify_summary.py",
  "verify:fast": "npm run lint && python -m pytest tests/python/ -m \"not live\"",
  "verify:live": "python -m pytest tests/python/ -m live -v",
  "verify:playwright": "npx playwright test",
  "lint": "npm run lint:py && npm run lint:js",
  "test:fast": "npm run lint && python -m pytest tests/python/ -m \"not live\""
}
```

Note: `test:fast` and `verify:fast` are intentionally identical — test:fast is the pre-push hook entry point, verify:fast is the first tier of the verify chain. Keeping both maintains backward compatibility with lefthook.yml which references test:fast.
