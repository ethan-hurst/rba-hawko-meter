Create `tests/python/test_zscore.py` with top-level test functions (no classes, per user decision). Use `@pytest.mark.parametrize` where natural.

**Functions under test** (all from `pipeline.normalize.zscore`):

1. `calculate_mad(values)` — MAD = median(|x_i - median(x)|) * 1.4826
   Tests:
   - Known-answer: `[1.0, 2.0, 3.0]` -> median=2.0, deviations=[1,0,1], median(deviations)=1.0, MAD=1.4826. Assert `pytest.approx(1.4826, rel=1e-4)`.
   - Known-answer: `[2.0, 4.0, 6.0, 8.0]` -> median=5.0, deviations=[3,1,1,3], median(deviations)=2.0, MAD=2.9652. Assert `pytest.approx(2.9652, rel=1e-4)`.
   - Edge case: all identical `[5.0, 5.0, 5.0]` -> MAD=0.0 exactly.
   - Edge case: single value `[3.0]` -> MAD=0.0 (deviation from self is 0).
   - Edge case: `[1.0, 1.0, 5.0]` -> median=1.0, deviations=[0,0,4], median(deviations)=0.0, MAD=0.0. This is a known edge case where majority-identical values yield zero MAD.
   - Use numpy arrays for all inputs.

2. `robust_zscore(current_value, window_values)` — z = (current - median) / MAD
   Tests:
   - Standard case: current=5.0, window=[1,2,3,4,5] -> compute median, MAD, and expected z manually.
   - Zero-MAD guard: window=[3.0, 3.0, 3.0], current=5.0 -> returns 0.0 (MAD=0 triggers guard).
   - Current equals median: current at median of window -> z=0.0.

3. `compute_rolling_zscores(df, window_quarters, min_quarters)` — rolling look-back window
   Tests:
   - Small dataset with `min_quarters=2`: first 2 rows have NaN z_score (window too small), row 3+ has valid z_score. Verify output has columns z_score, rolling_median, rolling_mad, window_size.
   - All-identical values in window: z_scores should be 0.0 (not NaN from division by zero).
   - Use the fixture_cpi_df with explicit `min_quarters=5` to test with realistic data. Verify first 5 rows have NaN z_scores and remaining rows have non-NaN z_scores.
   - Verify window_size column is correct (increments up to window_quarters cap).
   - Regression detection test: construct a known 10-row DataFrame where row 8 has a deliberate spike. With `min_quarters=3`, the z-score for row 8 should be significantly positive (> 1.5). This proves a regression in the z-score formula would be caught.

4. `determine_confidence(window_size)` — threshold-based classification
   Tests using `@pytest.mark.parametrize`:
   - window_size=32 -> "HIGH" (>= CONFIDENCE_HIGH_MIN_OBS=32)
   - window_size=40 -> "HIGH"
   - window_size=20 -> "MEDIUM" (>= CONFIDENCE_MEDIUM_MIN_OBS=20)
   - window_size=31 -> "MEDIUM"
   - window_size=19 -> "LOW"
   - window_size=0 -> "LOW"

**Floating-point tolerance:** Use `pytest.approx(expected, rel=1e-4)` for MAD/z-score results. Use `abs=1e-9` for cases where expected is 0.0.

**Data construction:** Use `numpy.array()` for direct function inputs. Use `pandas.DataFrame({"date": [...], "value": [...]})` with `pd.to_datetime` for compute_rolling_zscores inputs.

**Imports:** `import numpy as np`, `import pandas as pd`, `import pytest`, `from pipeline.normalize.zscore import calculate_mad, robust_zscore, compute_rolling_zscores, determine_confidence`.

**DO NOT** use test classes. **DO NOT** test engine.py functions. **DO NOT** import DATA_DIR directly.

Create `tests/python/test_gauge.py` with top-level test functions (no classes). Use `@pytest.mark.parametrize` extensively for boundary values.

**Functions under test** (all from `pipeline.normalize.gauge`):

1. `zscore_to_gauge(z, clamp_min, clamp_max)` — linear map z->gauge, clipped to [0,100]
   Tests using `@pytest.mark.parametrize`:
   - z=0.0 -> gauge=50.0 (midpoint)
   - z=-3.0 -> gauge=0.0 (clamp_min boundary)
   - z=3.0 -> gauge=100.0 (clamp_max boundary)
   - z=1.5 -> gauge=75.0
   - z=-1.5 -> gauge=25.0
   - z=4.0 -> gauge=100.0 (clamped above max)
   - z=-4.0 -> gauge=0.0 (clamped below min)
   - z=NaN -> gauge=NaN (NaN passthrough)
   - Use default clamp_min=-3.0, clamp_max=3.0 for all above.
   - Custom clamp test: clamp_min=-2.0, clamp_max=2.0, z=0.0 -> gauge=50.0 (midpoint unchanged).
   Use `pytest.approx` for float comparisons. For NaN test, use `math.isnan()`.

2. `classify_zone(gauge)` — 5-zone classification at exact boundaries
   Tests using `@pytest.mark.parametrize` (per user decision — test exact threshold values):
   - (0.0, "cold", "Strong dovish pressure")
   - (19.9, "cold", "Strong dovish pressure")
   - (20.0, "cool", "Mild dovish pressure") — boundary: < 20 is cold, >= 20 is cool
   - (39.9, "cool", "Mild dovish pressure")
   - (40.0, "neutral", "Balanced")
   - (50.0, "neutral", "Balanced")
   - (59.9, "neutral", "Balanced")
   - (60.0, "warm", "Mild hawkish pressure")
   - (79.9, "warm", "Mild hawkish pressure")
   - (80.0, "hot", "Strong hawkish pressure")
   - (100.0, "hot", "Strong hawkish pressure")
   - NaN -> ("unknown", "Insufficient data")
   Assert both zone_id and zone_label match.

3. `compute_hawk_score(gauge_values, weights, exclude_benchmark)` — weighted average with rebalancing
   Tests:
   - Single indicator: gauge_values={"inflation": 60.0}, weights={"inflation": {"weight": 0.25}} -> hawk_score=60.0.
   - Two indicators with equal weight: {"inflation": 40.0, "wages": 80.0} each weight 0.5 -> hawk_score=60.0.
   - Missing indicator rebalancing: weights define 3 indicators but gauge_values only has 2 -> weights rescaled so available sum to 1.0, producing correct weighted average.
   - All missing (empty gauge_values) -> returns 50.0 (neutral fallback).
   - Exclude benchmark: gauge_values includes "asx_futures" key but with exclude_benchmark=True, it's excluded from computation.
   - Boundary: result is always clamped to [0.0, 100.0].

4. `generate_verdict(hawk_score)` — threshold-based verdict strings
   Tests using `@pytest.mark.parametrize`:
   - hawk_score=10.0 -> "Economic indicators suggest strong easing pressure"
   - hawk_score=30.0 -> "Economic indicators suggest mild easing pressure"
   - hawk_score=50.0 -> "Economic indicators are broadly balanced"
   - hawk_score=70.0 -> "Economic indicators suggest moderate tightening pressure"
   - hawk_score=90.0 -> "Economic indicators suggest strong tightening pressure"
   - Boundary: 19.9 -> strong easing, 20.0 -> mild easing, 39.9 -> mild easing, 40.0 -> balanced, etc.

**Imports:** `import math`, `import pytest`, `from pipeline.normalize.gauge import zscore_to_gauge, classify_zone, compute_hawk_score, generate_verdict`.
