```bash
npm install --save-dev eslint@10 globals
```

This installs ESLint 10.0.2 (latest stable) and globals 17.3.0. The `@eslint/js` package ships bundled with ESLint v10 — no separate install needed. `jiti` is auto-installed as a peer dependency.

**Step 2: Create `eslint.config.js` at project root**

Use CommonJS format (no `"type": "module"` in package.json). Apply eslint:recommended via `js.configs.recommended` from `@eslint/js`.

```javascript
const js = require("@eslint/js");
const { defineConfig } = require("eslint/config");
const globals = require("globals");

module.exports = defineConfig([
  {
    files: ["public/js/**/*.js"],
    plugins: { js },
    extends: ["js/recommended"],
    languageOptions: {
      sourceType: "script",
      globals: {
        ...globals.browser,
        Plotly: "readonly",
        Decimal: "readonly",
        GaugesModule: "readonly",
        ChartModule: "readonly",
        DataModule: "readonly",
        CountdownModule: "readonly",
        CalculatorModule: "readonly",
        InterpretationsModule: "readonly",
      },
    },
  },
]);
```

**Step 2: Check remaining violations**

```bash
npx eslint public/js/
```

Review the output. Common violation types from eslint:recommended:
- `no-unused-vars`: Remove unused variables or prefix with `_`
- `no-undef`: Missing global declaration — add to eslint.config.js globals
- `no-redeclare`: Duplicate variable declarations
- `no-empty`: Empty catch blocks — add a comment explaining why empty
- `max-len`: Lines exceeding 88 chars — wrap using same strategies as Python

**Step 3: Fix remaining violations manually**

For each violation:
- `no-undef`: If it's a legitimate cross-file global, add it to `eslint.config.js` globals. If it's a typo or error, fix the code.
- `no-unused-vars`: Remove the unused variable if safe. If the variable is needed for a function signature (e.g., callback with unused parameter), prefix with `_`.
- `max-len` (if added): Wrap long lines. In JS, use template literals or string concatenation for long strings. Break long function calls after opening paren.
- Any other violations: Fix the actual code issue — do NOT use `// eslint-disable` comments (per CONTEXT.md: no inline suppressions).

IMPORTANT constraints:
- Do NOT add any `// eslint-disable` or `// eslint-disable-next-line` comments
- Do NOT change any functional behavior — only fix linting violations
- Match existing code style for semicolons and indentation
- If any fixes seem risky (changing behavior), investigate more carefully before applying
