Create `tests/python/test_live_sources.py` with 9 test functions — one per distinct `fetch_and_save()` call. All functions carry `@pytest.mark.live`. The existing `isolate_data_dir` autouse fixture already patches `DATA_DIR` to `tmp_path`, and `block_network` exempts `@pytest.mark.live` tests. No conftest.py changes needed.

**Pattern for each test function:**

1. Import the ingestor module
2. Call `fetch_and_save()` inside a try-except block
3. On success: validate structural requirements (required columns in CSV, required keys in return dict) — structural failures = hard assertion failure
4. On data quality issues (empty DataFrame, stale data >90 days): call `warnings.warn(msg, UserWarning, stacklevel=2)` with URL/reason details and return (test passes)
5. On exception (network error, timeout, HTTP error): call `warnings.warn()` with the error details and return (test passes)

**9 test functions:**

1. `test_live_abs_cpi` — calls `abs_data.fetch_and_save("cpi")`, checks `tmp_path / "abs_cpi.csv"` has columns `{date, value, source}`, warns if 0 rows or stale >90 days
2. `test_live_abs_employment` — calls `abs_data.fetch_and_save("employment")`, checks `abs_employment.csv` structure
3. `test_live_abs_wpi` — calls `abs_data.fetch_and_save("wage_price_index")`, checks `abs_wage_price_index.csv` structure
4. `test_live_abs_spending` — calls `abs_data.fetch_and_save("household_spending")`, checks `abs_household_spending.csv` structure
5. `test_live_abs_building_approvals` — calls `abs_data.fetch_and_save("building_approvals")`, checks `abs_building_approvals.csv` structure
6. `test_live_rba_cash_rate` — calls `rba_data.fetch_and_save()`. NOTE: rba_data.fetch_and_save() returns `int` (row count) and CAN raise exceptions (it does not wrap in try-except like scrapers). Wrap the call in try-except. On success, check that `tmp_path / "rba_cash_rate.csv"` exists and has columns `{date, value, source}`. On exception, warn.
7. `test_live_asx_futures` — calls `asx_futures_scraper.fetch_and_save()`, returns status dict. On `status != "success"`, warn. On success, check `tmp_path / "asx_futures.csv"` exists with expected columns `{date, meeting_date, implied_rate, probability_cut, probability_hold, probability_hike}`.
8. `test_live_cotality` — calls `corelogic_scraper.fetch_and_save()`, returns status dict. On `status != "success"`, warn with error detail. On success, check `tmp_path / "corelogic_housing.csv"` exists with expected structure.
9. `test_live_nab_capacity` — calls `nab_scraper.fetch_and_save()`, returns status dict. Same pattern as Cotality.

**Important implementation details:**
- Use `warnings.warn(msg, UserWarning, stacklevel=2)` — NOT `pytest.warns()`. The test must pass whether the endpoint is up or down. `pytest.warns()` would FAIL when the endpoint works correctly (no warning emitted).
- For ABS tests (1-5), the fetch_and_save(series) returns `{series: row_count}`. The CSV is written to `DATA_DIR / config["output_file"]` which is `tmp_path / filename` due to autouse fixture.
- For RBA test (6), fetch_and_save() returns int and can raise. Must try-except the entire call.
- For scrapers (7-9), fetch_and_save() returns status dict and NEVER raises (they handle exceptions internally).
- Staleness check: read CSV, parse date column, warn if latest date is >90 days old. Only check on successful fetch (don't check on warn path).
- Each warning message should include: source name, what went wrong, and any available HTTP context.
- No test classes — all top-level test functions (project convention from Phase 12).
- Import `pipeline.config` as a module inside the test function body (after the autouse fixture has patched DATA_DIR), OR rely on the autouse fixture having already patched it (imports at top are fine since `fetch_and_save` reads `DATA_DIR` at call time, not import time — per Phase 11 decision).

**Part A: Create `scripts/verify_summary.py`**

Python script that reads `public/data/status.json` (after `pipeline/main.py` has run) and prints an ASCII box-drawing table. This is NOT called by pytest — it is called by `npm run verify` after the pipeline finishes.

Implementation:
1. Read `public/data/status.json` using `json.load()`. If file does not exist, print error and exit 1.
2. Extract `status["gauges"]` (7 gauge keys) and `status["overall"]["hawk_score"]`.
3. Check that all 7 expected gauge keys are present: `inflation`, `wages`, `employment`, `spending`, `building_approvals`, `housing`, `business_confidence`.
4. Check `hawk_score` is in range `0 <= hawk_score <= 100`.
5. For each gauge, determine status:
   - Key missing from gauges = `FAIL`
   - `staleness_days > 90` = `WARN` (data may be stale)
   - Otherwise = `PASS`
6. Format latest data date from `data_date` field as `Mon YYYY` (e.g., "Jan 2026"). Handle missing/invalid dates gracefully.
7. Print ASCII table using box-drawing `─` characters:

```
────────────────────────────────────────────────
  RBA Hawk-O-Meter — Live Verification
────────────────────────────────────────────────
 Indicator              Status   Latest
────────────────────────────────────────────────
 ABS CPI                PASS     Jan 2026
 ABS Employment         PASS     Jan 2026
 ABS WPI                PASS     Oct 2025
 ABS Household Spending PASS     Jan 2026
 ABS Building Approvals PASS     Jan 2026
 Cotality HVI           WARN     Feb 2026
 NAB Capacity           PASS     Feb 2026
────────────────────────────────────────────────
 Hawk Score: 62 / 100
 Result: PASS (1 warning)
────────────────────────────────────────────────
```

8. Exit codes:
   - Missing indicator keys = exit 1 (`FAIL`)
   - hawk_score out of [0, 100] = exit 1
   - All keys present with warnings only = exit 0 (`PASS` with warnings)
   - All keys present, no warnings = exit 0 (`PASS`)

Label mapping (gauge key -> display name):
```python
INDICATOR_LABELS = {
    "inflation":          "ABS CPI",
    "employment":         "ABS Employment",
    "wages":              "ABS WPI",
    "spending":           "ABS Household Spending",
    "building_approvals": "ABS Building Approvals",
    "housing":            "Cotality HVI",
    "business_confidence":"NAB Capacity",
}
```

Do NOT re-validate the full JSON schema (unit tests own that in test_schema.py). Only check for 7 gauge keys + hawk_score range.
