Add three functions to the InterpretationsModule IIFE, placed BEFORE the return statement (after the existing `getPlainVerdict` function around line 566):

**1. `rankIndicators(gaugesData)` — internal helper (NOT exported):**
- Iterate over `Object.keys(gaugesData)`, for each metric:
  - Skip if `m.value == null` (null data guard)
  - Skip if `isDataSuspect(metricId, m)` returns true (data quality guard)
  - Compute `contribution = (m.value - 50) * m.weight`
  - If `contribution >= 0.5`: push to hawkish array as `{ id: metricId, data: m, contribution: contribution }`
  - If `contribution <= -0.5`: push to dovish array as `{ id: metricId, data: m, contribution: contribution }`
  - Otherwise: omit (neutral, below threshold — satisfies EXPL-03)
- Sort hawkish descending by contribution: `b.contribution - a.contribution`
- Sort dovish by contribution ascending (most negative first): `a.contribution - b.contribution`
- Return `{ hawkish: hawkish.slice(0, 3), dovish: dovish.slice(0, 2) }`

**2. `getExplanationSentence(metricId, metricData)` — public function:**
- Extract `raw` value: `metricData.raw_value != null ? parseFloat(metricData.raw_value).toFixed(1) : null`
- Extract `v` = `metricData.value`
- Switch on metricId with hawkish (v > 60) and dovish (v < 50) variants for each. Use the EXACT sentence templates from CONTEXT.md (section 3):

  **inflation (v < 50):** `'Inflation at ' + raw + '% per year is within the RBA\u2019s target range, which has historically been associated with less pressure to raise interest rates.'`
  **inflation (v >= 50):** `'Inflation at ' + raw + '% per year is above the RBA\u2019s target range, which tends to increase pressure on interest rates.'`

  **wages (v > 60):** `'Wages growing at ' + raw + '% per year tends to push up costs and prices, which has historically been associated with upward pressure on rates.'`
  **wages (v <= 60):** `'Wage growth at ' + raw + '% per year is subdued, which tends to reduce upward pressure on prices and interest rates.'`

  **employment (v > 60):** `'The job market is very tight, with strong demand for workers, which tends to push up wages and prices.'`
  **employment (v <= 60):** `'The job market is softening, which has historically been associated with less pressure on interest rates.'`

  **housing (v > 60):** `'Housing prices are rising at ' + raw + '% per year, which has historically been associated with upward pressure on interest rates.'`
  **housing (v <= 60):** `'Housing price growth has slowed, which tends to reduce pressure on interest rates.'`

  **spending (v > 60):** `'Consumer spending is running above trend, which tends to add to price pressures.'`
  **spending (v <= 60):** `'Consumer spending is subdued, which has historically been associated with less inflationary pressure.'`

  **building_approvals (v > 60):** `'Building approvals are above average, which is consistent with strong demand in the construction sector.'`
  **building_approvals (v <= 60):** `'Building approvals are below average, which tends to signal a slowing construction sector.'`

  **business_confidence (v > 60):** `'Capacity utilisation at ' + raw + '% is above the long-run average, which tends to signal inflationary pressure.'`
  **business_confidence (v <= 60):** `'Capacity utilisation is below average, which has historically been associated with reduced pressure on prices.'`

  **default:** `'This indicator is currently ' + (v > 50 ? 'above' : 'below') + ' its historical average.'`

- Note: The hawkish/dovish threshold for sentence selection is separate from the ranking threshold. Use v > 60 for hawkish sentences, v < 50 for dovish sentences (matching the zone boundaries in context). For values 50-60, use the hawkish/above variant as it's the mildly-above-neutral case.

**3. `renderVerdictExplanation(containerId, data)` — public function:**
- Get container: `var container = document.getElementById(containerId);`
- Guard: `if (!container || !data || !data.gauges) return;`

- Call `rankIndicators(data.gauges)` to get `ranked`
- **All-neutral edge case:** If both `ranked.hawkish.length === 0` AND `ranked.dovish.length === 0`:

If yes: find top 1 indicator from the dominant direction (hawkish if score > 60, dovish if score < 40) by removing threshold filter, show it with softer language prefix: `'The main contributor is... '` + getExplanationSentence() + `', though no single indicator is applying strong pressure.'`
  - Otherwise: create a `<p>` with class `text-sm text-gray-400` and textContent: `'No indicators are currently applying significant pressure in either direction, which is consistent with the balanced reading.'`
  - Return early after appending.
- **Hawkish section (if ranked.hawkish.length > 0):**
  - Create `<h3>` with className `'text-base font-semibold mb-2'`
  - Set `h3.style.color = GaugesModule.getZoneColor(75)` (yields '#f87171')
  - Set textContent `'Pushing the score up'`

  - Create `<ul>` with className `'list-disc list-inside mb-4 space-y-1'`

  - Append ul to container
