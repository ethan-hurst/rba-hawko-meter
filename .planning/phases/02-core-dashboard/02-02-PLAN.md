  <name>Task 1: Create data fetching and chart modules</name>
  <files>

  </files>
  <action>
    Create the data layer and Plotly chart module using IIFE patterns for encapsulation.

    **public/js/data.js -- DataModule:**

    IIFE pattern exposing a public API. Responsibilities:
    - `fetch(url)` -- Fetch JSON from URL with caching (Map-based cache)
    - `showError(containerId, message)` -- Display styled error in a container using safe DOM methods (createElement/textContent, NOT innerHTML)
    - `showLoading(containerId, message)` -- Display loading spinner/text in a container
    - Cache: Simple Map keyed by URL, cleared on page unload
    - Error handling: catch fetch failures, throw descriptive Error with URL

    ```javascript
    const DataModule = (() => {
      // Private cache, public fetch/showError/showLoading
    })();
    ```

    **public/js/chart.js -- ChartModule:**

    IIFE pattern exposing chart creation API. Responsibilities:

    1. `createDarkLayout()` -- private helper returning Plotly layout object:
       - template: 'plotly_dark'
       - paper_bgcolor: '#0a0a0a' (finance-dark)
       - plot_bgcolor: '#0a0a0a'
       - font: { color: '#e5e5e5', family: 'Inter, system-ui, sans-serif' }
       - xaxis: gridcolor '#2d2d2d', type 'date', rangeslider visible false
       - yaxis: gridcolor '#2d2d2d', title 'Cash Rate (%)', ticksuffix '%'
       - hovermode: 'x unified'
       - margin: { t: 40, r: 20, b: 60, l: 60 }

    2. `createTimeframeButtons()` -- private helper returning rangeselector config:
       - Buttons: 1Y, 5Y, 10Y, All
       - Styled for dark theme: bgcolor '#2d2d2d', activecolor '#60a5fa', font color '#e5e5e5'

    3. `addRateChangeAnnotations(rateChanges)` -- private helper:
       - For each rate change, create a vertical line shape:
         * Red (#ef4444) dotted line for direction "up" (hikes)
         * Green (#10b981) dotted line for direction "down" (cuts)
         * yref: 'paper', y0: 0, y1: 1 (full height)
       - For each rate change, create an annotation:
         * Text: "+0.25%" or "-0.25%" showing change amount
         * Position at top of chart (y: 0.95, yref: 'paper')
         * Colored to match direction
         * Small font (11px), dark background for readability
       - Return { shapes, annotations }

    4. `create(containerId, rateData)` -- public method:
       - Build trace: scatter line chart with dates on x, rates on y
       - Line color: '#60a5fa' (finance-accent), width 2
       - Hover template: '%{y:.2f}%<extra>RBA Cash Rate</extra>'
       - Build layout: merge createDarkLayout() + createTimeframeButtons() on xaxis
       - Add rate change annotations from rateData.rate_changes
       - Config: responsive true, displaylogo false, displayModeBar true
       - Remove mode bar buttons: select2d, lasso2d, autoScale2d
       - Call Plotly.newPlot(containerId, [trace], layout, config)

    5. `resize(containerId)` -- public method:
       - Call Plotly.Plots.resize(containerId)
       - Used for orientation changes

    **Important from research:**
    - Always use `responsive: true` in config
    - Don't set fixed width/height in layout (let container control)
    - Use safe DOM methods throughout (no innerHTML)
    - Rate change annotations make the chart much more informative -- this is a locked decision from CONTEXT
  </action>

<task type="auto">
  <name>Task 2: Create countdown timer, main initialization, and deployment config</name>
  <files>
    public/js/countdown.js
    public/js/main.js
    netlify.toml
  </files>
  <action>
    Create the countdown timer module, main initialization module, and Netlify config.

    **public/js/countdown.js -- CountdownModule:**

    IIFE pattern with interval cleanup. Responsibilities:

    1. `calculateTimeRemaining(targetDate)` -- private:
       - Calculate difference between now and target
       - Return { days, hours, minutes, seconds, isPast, isToday }
       - isToday: check if target date is same calendar day in Australia/Sydney timezone
       - isPast: difference <= 0

    2. `updateDisplay(containerEl, time)` -- private:
       - Use safe DOM methods (createElement/textContent only)
       - If isToday: show highlight banner
         * "RBA meeting TODAY -- decision expected 2:30pm AEST"
         * Styled: amber/gold background border, bold text
       - If isPast (and not today -- meeting has passed):
         * "RBA Board met on [date]. Next meeting: [next date]"
       - Otherwise: show live countdown
         * "Next RBA Board Meeting" heading
         * Display meeting date: "Tuesday 3 March 2026, 2:30pm AEST"
         * Live counter: "14 days  3 hours  22 minutes  48 seconds"
         * Each unit in its own styled box/span with label below
         * Use Tailwind-compatible classes for dark theme styling

    3. `start(containerEl, meetingData)` -- public:
       - Clear any existing interval (prevent memory leaks)
       - Parse target date: new Date(meetingData.next_meeting.date)
       - Set up setInterval at 1000ms
       - Call updateDisplay immediately (don't wait 1s for first render)
       - Store intervalId for cleanup

    4. `stop()` -- public:
       - Clear interval if running

    5. Cleanup: `window.addEventListener('beforeunload', stop)` -- registered once inside IIFE

    **public/js/main.js -- Dashboard initialization:**

    DOMContentLoaded handler that orchestrates all modules. Responsibilities:

    1. Show loading states in chart and countdown containers
    2. Fetch data in parallel:
       - DataModule.fetch('data/rates.json')
       - DataModule.fetch('data/meetings.json')
       - Use Promise.allSettled (not Promise.all) so one failure doesn't block the other
    3. If rates.json loaded successfully:
       - Update the cash rate display element with current_rate value
       - Update the "last updated" text
       - Update source citation
       - Call ChartModule.create('rate-chart', rateData)
    4. If rates.json failed:
       - Show error state in chart container and cash rate display
    5. If meetings.json loaded successfully:
       - Call CountdownModule.start(countdownEl, meetingData)
    6. If meetings.json failed:
       - Show error in countdown container
    7. Set up resize handler:
       - On window resize, debounce and call ChartModule.resize('rate-chart')
       - Use a 250ms debounce to prevent excessive redraws
    8. Dismiss banner handler:
       - Add click handler to disclaimer banner close button
       - Hide banner (display: none), store in sessionStorage so it stays dismissed during session

    **netlify.toml:**

    ```toml
    [build]
      publish = "public"

    [[headers]]
      for = "/*"
      [headers.values]
        X-Frame-Options = "DENY"
        X-Content-Type-Options = "nosniff"
        Referrer-Policy = "strict-origin-when-cross-origin"

    [[headers]]
      for = "/data/*"
      [headers.values]
        Cache-Control = "public, max-age=3600"
    ```

    - publish directory is "public" (where index.html lives)
    - Security headers for good practice
    - Data files cached for 1 hour (weekly pipeline updates)

    **Important from research:**
    - Always clear intervals before creating new ones
    - Use window.addEventListener('beforeunload', ...) for cleanup
    - Store dates in ISO 8601 with timezone offset
    - Use Intl.DateTimeFormat with timeZone: 'Australia/Sydney' for display
    - Promise.allSettled prevents cascade failures
  </action>
