    Create RBA data ingestor that downloads cash rate from RBA statistical tables:

    **pipeline/ingest/rba_data.py:**

    - Import from pipeline.config, pipeline.utils.csv_handler, pipeline.utils.http_client
    - `fetch_cash_rate() -> pd.DataFrame`:
      * Create session via create_session()
      * Download RBA Table A2 CSV from config URL (cash rate target)
      * RBA CSVs have metadata rows at the top -- use skiprows to skip header metadata (typically ~10 rows, but detect dynamically by finding the row with "Date" or "Series ID")
      * Parse the date column using dayfirst=True (Australian format DD-Mon-YYYY like "02-Jan-2020")
      * Extract date and cash rate target value columns
      * Rename to standardized columns: date, value
      * Add 'source' column = 'RBA'
      * Return DataFrame

    - `fetch_and_save() -> int`:
      * Call fetch_cash_rate()
      * Call append_to_csv(config path for rba_cash_rate.csv, df)
      * Return row count

    - `if __name__ == '__main__':` block:
      * Call fetch_and_save()
      * Print result summary

    **Important notes from research:**
    - RBA CSV format: metadata rows at top, then column headers, then data
    - Date format is DD-Mon-YYYY (e.g., "02-Jan-2020")
    - Use pd.to_datetime with dayfirst=True
    - Store dates as ISO 8601 in output CSV
    - The actual URL pattern is: https://www.rba.gov.au/statistics/tables/csv/a2-daily.csv (note the "-daily" suffix for table A2)
    - Table A2 contains "Cash Rate Target" -- extract this specific column
  </action>

  <name>Task 3: Create ABS economic data ingestors</name>
  <files>
    pipeline/ingest/abs_data.py
  </files>
  <action>
    Create ABS data ingestor using the ABS Data API (SDMX 2.1):

    **pipeline/ingest/abs_data.py:**

    - Import from pipeline.config, pipeline.utils.csv_handler, pipeline.utils.http_client

    - `fetch_abs_series(dataflow_id: str, key: str, params: dict = None) -> pd.DataFrame`:
      * Core function for all ABS data retrieval
      * Create session via create_session()
      * Build URL: f"{ABS_API_BASE}/ABS,{dataflow_id}/{key}"
      * Set Accept header: 'application/vnd.sdmx.data+csv;labels=both'
      * GET request with timeout from config
      * Parse CSV response with pandas
      * Rename TIME_PERIOD -> date, OBS_VALUE -> value
      * Convert date to datetime (handle both YYYY-MM and YYYY-QN quarterly formats)
      * Add source column = 'ABS'
      * Add series_id column for traceability
      * Return DataFrame[date, value, source, series_id]
      * If request fails, raise with clear error message including URL

    - Individual fetchers (each calls fetch_abs_series with correct params):
      * `fetch_cpi() -> pd.DataFrame` -- Consumer Price Index, All Groups, Weighted Average of 8 Capital Cities. Quarterly data. Use startPeriod param to limit to last 10 years.
      * `fetch_employment() -> pd.DataFrame` -- Labour Force total employment and unemployment rate. Monthly data.
      * `fetch_retail_trade() -> pd.DataFrame` -- Retail Trade total turnover. Monthly data.
      * `fetch_wage_price_index() -> pd.DataFrame` -- Wage Price Index, total hourly rates of pay excluding bonuses. Quarterly data.
      * `fetch_building_approvals() -> pd.DataFrame` -- Building Approvals, total dwellings. Monthly data.

    - `FETCHERS` dict mapping name -> (fetch_function, output_csv_filename):
      * 'cpi': (fetch_cpi, 'abs_cpi.csv')
      * 'employment': (fetch_employment, 'abs_employment.csv')
      * 'retail_trade': (fetch_retail_trade, 'abs_retail_trade.csv')
      * 'wage_price_index': (fetch_wage_price_index, 'abs_wage_price_index.csv')
      * 'building_approvals': (fetch_building_approvals, 'abs_building_approvals.csv')

    - `fetch_and_save(series: str = None) -> dict`:
      * If series specified, run just that fetcher
      * If series is None, run all fetchers
      * For each: call fetcher, append_to_csv, collect results
      * Return dict of {name: row_count}

    - `if __name__ == '__main__':` block:
      * Accept optional CLI arg for single series (e.g., `python -m pipeline.ingest.abs_data cpi`)
      * Default: run all
      * Print summary

    **Important notes from research:**
    - ABS Data API base: https://data.api.abs.gov.au/data
    - Request CSV format via Accept header (not XML)
    - SDMX URL format: /data/ABS,{dataflow}/{key}?startPeriod=YYYY&detail=dataonly
    - Quarterly dates come as "2024-Q1" format -- convert to first day of quarter
    - Monthly dates come as "2024-01" format -- convert to first day of month
    - The exact dataflow IDs and series keys need to be determined at implementation time by inspecting the ABS API. Common patterns:
      * CPI: dataflow "CPI", key like "1.10001.10.Q" (Index Number, All groups CPI, Original)
      * LF (Labour Force): "M1.1.AUS" style keys
    - If exact keys are unclear, the API supports wildcards -- use key="all" with query params to filter
    - Add startPeriod=2014 (or similar) to limit to ~10 years of data
  </action>
