<task type="auto">
  <name>Task 1: Create pyproject.toml config hub with pytest and ruff sections</name>
  <files>pyproject.toml</files>
  <action>
Create `pyproject.toml` at the project root with two tool sections:

**[tool.pytest.ini_options]:**
- `minversion = "6.0"` — minimum compatible pytest version
- `testpaths = ["tests/python"]` — scope test discovery to Python tests only (keeps Playwright tests in `tests/*.spec.js` separate)
- `pythonpath = ["."]` — adds project root to sys.path so `import pipeline.config` works from test files (flat layout, no `src/`)
- `addopts = ["--strict-markers"]` — typo in `@pytest.mark.X` raises error instead of silently creating a new marker
- `markers` array with one entry: `"live: marks tests that require live network access (deselect with '-m \"not live\"')"` — registers the `live` marker for FOUND-04 tier separation

**[tool.ruff]:**
- `target-version = "py313"` — match the project's Python version
- `line-length = 88` — ruff default, standard for Python projects
- `exclude` array: `.git`, `.venv`, `__pycache__`, `node_modules`, `.pytest_cache`

**[tool.ruff.lint]:**
- `select = ["E", "F", "W", "B", "I", "UP"]` — pycodestyle errors, pyflakes, warnings, bugbear, isort, pyupgrade
- `ignore = []` — start clean, no suppressions
- `fixable = ["ALL"]` — allow auto-fixing for all selected rules

Do NOT include a `[project]` section — this project is not a distributable Python package. The pyproject.toml is purely a tool configuration hub.
  </action>
