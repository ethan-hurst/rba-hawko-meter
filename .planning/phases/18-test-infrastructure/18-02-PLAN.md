<task type="auto">
  <name>Task 2: Create per-module coverage check script</name>
  <files>scripts/check_coverage.py</files>
  <action>
Create `scripts/check_coverage.py` — a standalone Python script that reads `.coverage.json` (written by pytest-cov per Plan 18-01), extracts per-module coverage for `pipeline/` files, and enforces a minimum threshold.

**Implementation requirements (from locked decisions):**
1. Accept `--min` flag (default 85.0) for the minimum coverage percentage.
2. Read `.coverage.json` from project root. If not found, print `"ERROR: .coverage.json not found. Run pytest first."` and exit 1.
3. Parse JSON: `data["files"]` is a dict mapping file paths to info objects. Each info has `summary.percent_covered` (float) and `summary.num_statements` (int).
4. Filter to files starting with `pipeline/` where `num_statements > 0` — this excludes `__init__.py` files with no executable code.
5. Compare each module's `percent_covered` against `--min` threshold.
6. **On failure (any module below threshold):** Print a diff table with columns: Module, Actual, Target, Gap. Module = filename only (not full path). Gap = actual - target (negative number). Color rows RED when outputting to terminal. Exit 1.
7. **On success (all modules at or above threshold):** Print a summary table showing each module and its coverage %. Color rows GREEN when outputting to terminal. Exit 0.
8. **Color auto-detect:** Use `sys.stdout.isatty()`. ANSI colors when True, plain text when False. No third-party color library — use stdlib only (YAGNI).
9. Add `#!/usr/bin/env python3` shebang and module docstring.

**Table format (from locked decision):**
- Failure diff table:
  ```
  Coverage below 85% threshold:

    Module                                    Actual   Target      Gap
    ---------------------------------------- -------  -------  -------
    abs_data.py                                  72%      85%     -13%
  ```
- Success summary table:
  ```
  All pipeline/ modules at or above 85%:

    abs_data.py                                  92%
  ```

Do NOT add `--cov-fail-under` to pyproject.toml — per-module enforcement is this script's job, not pytest-cov's global threshold.
  </action>
